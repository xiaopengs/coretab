# CoreTab — 产品介绍

> 下一代智能新标签页，让浏览器标签页管理变得优雅高效

---

## 产品愿景

CoreTab 重新定义浏览器新标签页的价值——它不仅是空白开始页，更是**标签页控制中心**和**内容发现入口**。

我们相信：用户值得拥有一个**整洁、高效、美观**的新标签页，而不是被臃肿的默认页面或杂乱的标签堆所困扰。

---

## 核心价值主张

### 1. 标签页管理 — 从混乱到有序

**问题**：打开 20+ 标签页后，浏览器变卡，查找困难。

**解决**：CoreTab 按窗口和域名智能分组，让标签页一目了然。

```
Before:  [🔵][🔵][🔵][🔵][🔵][🔵][🔵][🔵][🔵][🔵]... (混乱)

After:   ┌─────────────────┐
         │ GitHub     5tabs│  ┌─────────────────┐
         │ [repo][issue]   │  │ YouTube   3tabs │
         │ Gmail      3tabs│  │ [video][video]  │
         └─────────────────┘  └─────────────────┘
```

### 2. Closed Tabs — 后悔药

**问题**：误关闭标签页后，找不回来了。

**解决**：CoreTab 自动记录所有关闭的标签，按日期和网站分组，随时可恢复。

- 关闭即记录 — 不用担心丢失
- 日期分组 — 轻松找到之前的标签
- 一键恢复 — 点击即回，无需翻历史记录

### 3. GitHub Trending — 发现新大陆

**问题**：想跟进 GitHub 热门项目，但懒得专门去刷 Trending 页面。

**解决**：热门项目就在新标签页，每天打开都有新发现。

### 4. Clay/Swatch 设计 — 养眼舒适

**问题**：大多数扩展界面要么太暗要么太刺眼。

**解决**：温暖的奶油色背景 + 柔和的燕麦边框 + 精心调校的配色，长时间使用也不累。

## 设计哲学

### 1. 效率优先

每个交互都应该**零思考**：
- 关闭标签 → 点击即关，无需确认（域名关闭）
- 恢复标签 → 点击即开，立即可用
- 查看分组 → 一目了然，无需猜测

### 2. 信息分层

按照**注意力优先级**呈现内容：

| 层级 | 内容 | 展示方式 |
|------|------|----------|
| 第一层 | Open Tabs | 始终可见，最新状态 |
| 第二层 | Closed Tabs | 次要位置，可恢复 |
| 第三层 | GitHub Trending | 卡片网格，适合浏览 |
| 第四层 | History | 需要时展开 |

### 3. 渐进式披露

不一次性展示所有信息，而是**按需获取**：
- 标签页默认显示前 5 条，hover 显示关闭按钮
- History 默认显示 3 条，点击 Show more 展开
- 域名下超过 8 个标签时，显示 "+N more"

### 4. 即时反馈

每个操作都有**明确的视觉/文字反馈**：
- Toast 通知显示操作结果
- 按钮悬停有动画响应
- 关闭后立即刷新列表

## 用户旅程

### 场景一：整理工作标签

> 用户有 15 个工作标签分布在 3 个窗口

1. 打开新标签页，所有窗口标签一目了然
2. 发现 GitHub 窗口有 8 个标签，可以关闭不用的
3. 点击 "Close 3 tabs" 清理无用标签
4. 已关闭的标签自动进入 Closed Tabs，随时可恢复

### 场景二：恢复误关闭的标签

> 用户不小心关闭了一个重要的 GitHub issue 页面

1. 打开新标签页
2. 查看 Closed Tabs 区域
3. 找到 GitHub 分组下的该标签
4. 点击恢复，在新标签页打开

### 场景三：发现有趣项目

> 用户每天打开新标签页都想看看有什么好项目

1. 新标签页底部展示 GitHub Trending
2. 2 列卡片网格，快速浏览
3. 点击跳转到 GitHub 页面

## 技术架构

### 数据流

```
用户操作 → 事件处理 → 数据层 → 渲染层 → UI 更新
    ↓
chrome.tabs API → localStorage → DOM 更新
```

### 关键设计

1. **单一事件委托** — 全局一个 click 监听器，避免事件冲突
2. **URL 匹配** — 用 URL 而非 tabId 操作标签，更可靠
3. **本地优先** — 所有数据存储在 localStorage，保护隐私
4. **即时刷新** — 操作后立即更新 UI，无需等待

### 性能考量

- **懒加载** — GitHub Trending 只在需要时请求
- **缓存策略** — Trending 缓存 5 分钟
- **按需渲染** — 只渲染可见区域的内容

## 关于设计

CoreTab 采用 **Clay Design** 风格：

> Clay = 温暖 + 柔和 + 有触感

不是扁平化的冷漠，也不是拟物的厚重，而是一种**温暖、有呼吸感**的设计语言。

色彩取自自然：奶油、燕麦、绿叶、粉樱。
