# CoreTab 需求说明书 (PRD)

## 1. 项目概述

### 1.1 产品简介

| 项目 | 内容 |
|------|------|
| 产品名称 | CoreTab - 标签页管理器 |
| 产品类型 | Chrome 浏览器扩展插件 |
| 产品定位 | 帮助用户管理和保存 Chrome 标签页的工具 |
| 核心价值 | 将打开的标签页保存为可管理的链接列表，释放浏览器内存，随时一键恢复 |

### 1.2 产品愿景

让标签页管理变得简单高效，帮助用户摆脱"标签页爆炸"的困扰，同时不丢失任何重要的工作进度。

---

## 2. 用户分析

### 2.1 目标用户

| 用户群体 | 典型场景 |
|----------|----------|
| 知识工作者 | 同时打开多个文档、资料页面进行研究 |
| 开发者 | 打开大量技术文档、代码参考页面 |
| 学生 | 同时查阅多个学习资料、论文 |
| 设计师 | 收集大量参考图片、设计资源 |
| 普通用户 | 日常浏览时积累大量标签页 |

### 2.2 用户痛点

| 痛点 | 描述 | 严重程度 |
|------|------|----------|
| 标签页过多 | 打开几十个标签页，浏览器变卡 | 高 |
| 不舍关闭 | 想关闭浏览器但不舍得关闭正在看的页面 | 高 |
| 临时清理 | 临时需要清理标签页但不想丢失工作进度 | 中 |
| 跨设备访问 | 想在手机或其他设备上查看电脑上的标签页 | 低 |
| 标签页丢失 | 意外关闭浏览器后标签页丢失 | 中 |

## 3. 功能需求

### 3.1 核心功能 (MVP)

#### F1: 一键保存所有标签页
- **描述**: 用户点击按钮，一次性保存当前所有打开的标签页，并关闭原窗口所有标签页
- **触发方式**: 点击扩展图标
- **数据采集**: 标题、URL、favicon、保存时间
- **过滤规则**: 排除 chrome://、chrome-extension://、about: 开头的系统页面
- **去重逻辑**: 相同URL不重复保存
- **操作流程（关键）**:
  1. 获取当前焦点窗口（不干涉其他浏览器窗口）
  2. 检查窗口中是否存在多个 CoreTab，只保留一个
  3. 如果没有 CoreTab，先在当前窗口打开 CoreTab 管理页面
  4. 保存所有有效标签页到存储
  5. 关闭当前窗口中除 CoreTab 外的所有标签页（包括首页、其他扩展页面等）
  6. 刷新并激活 CoreTab 管理页面
- **多 CoreTab 处理**:
  - 如果窗口中存在多个 CoreTab，关闭多余的，只保留第一个
  - 刷新保留的 CoreTab 以显示最新数据
- **窗口隔离**:
  - 所有操作仅在当前焦点窗口进行
  - 不关闭、不干涉其他浏览器窗口的标签页

#### F2: 按日期分组展示
- **描述**: 在 CoreTab 管理页面中，按保存日期/时间分组展示标签页
- **分组格式**: 
  - 今天: "今天 HH:MM"
  - 昨天: "昨天 HH:MM"
  - 更早: "YYYY/MM/DD HH:MM"
- **显示内容**: 
  - 每组标题: "X 个标签页" + 保存时间
  - 标签项: favicon + 标题
  - 操作按钮: "全部还原"、"更多..."
- **排序方式**: 最新保存的组在前

#### F3: 恢复单个标签页
- **描述**: 点击列表中的标签项，快速重新打开该页面
- **打开方式**: 在新标签页中打开
- **反馈**: 标签项变为灰色/消失，表示已恢复

#### F4: 恢复整组标签页
- **描述**: 点击"全部还原"按钮，一次性恢复该组所有标签页
- **打开方式**: 在新窗口中打开所有标签页
- **反馈**: 该组从列表中移除或标记为已恢复

#### F5: 删除单个标签页
- **描述**: 点击删除按钮，移除已保存的标签页
- **确认**: 无需确认，直接删除
- **反馈**: 删除后更新列表

#### F6: 删除整组标签页
- **描述**: 通过"更多..."菜单删除整组保存的标签页
- **确认**: 需要二次确认
- **反馈**: 该组从列表中移除

#### F7: 管理页面独立显示
- **描述**: CoreTab 管理页面作为独立标签页打开，而非弹窗
- **页面URL**: chrome-extension://[id]/tabs.html
- **页面功能**:
  - 显示所有按日期分组的标签页
  - 支持搜索已保存的标签页
  - 支持导入/导出功能
  - 设置选项

#### F8: 相对时间显示
- **描述**: 显示人性化的相对时间
- **格式**: 
  - 1分钟内: "刚刚"
  - 1小时内: "X分钟前"
  - 24小时内: "X小时前"
  - 7天内: "X天前"
  - 更早: "YYYY/MM/DD"

#### F9: 分组命名
- **描述**: 允许用户给分组自定义名称
- **默认名称**: 使用保存时间
- **操作**: 点击分组名称可编辑

#### F10: 分享功能
- **描述**: 将分组转换为可分享的网页链接
- **生成**: 创建包含所有链接的 HTML 页面
- **用途**: 分享给他人或备份

#### F11: 锁定分组
- **描述**: 重要分组可锁定，防止误删
- **标识**: 锁定分组显示锁定图标
- **保护**: 锁定分组无法删除，需先解锁

### 3.2 高级功能 (v1.2.0+)

#### F8: 多会话管理
- **描述**: 保存多组标签页会话
- **功能**:
  - 创建新会话
  - 命名会话
  - 切换会话
  - 删除会话

#### F9: 会话导出/导入
- **描述**: 将会话导出为JSON文件，或从文件导入
- **用途**: 备份、分享

#### F10: 标签页同步 (v2.0)
- **描述**: 通过云端同步标签页到其他设备
- **技术方案**: Chrome Sync API 或 自建后端

#### F11: 搜索功能 (v2.0)
- **描述**: 通过关键词搜索已保存的标签页

## 4. 非功能需求

### 4.1 性能要求

| 指标 | 要求 |
|------|------|
| 首次加载 | < 500ms |
| 保存操作 | < 1s |
| 列表渲染 | 100个标签页 < 200ms |
| 内存占用 | < 50MB |

### 4.2 兼容性要求

| 项目 | 要求 |
|------|------|
| Chrome 版本 | Chrome 88+ (Manifest V3) |
| 浏览器 | Chrome, Edge, Brave 等 Chromium 内核浏览器 |

### 4.3 数据安全

- 所有数据存储在用户本地 (chrome.storage.local)
- 不上传任何数据到服务器
- 不收集任何用户行为数据

- 主色: #1a73e8 (Google Blue)
- 背景: #f8f9fa (浅灰)
- 卡片背景: #ffffff
- 文字: #333333
- 次要文字: #666666

## 6. 验收标准

### 6.1 MVP 验收

| 功能 | 验收条件 |
|------|----------|
| F1 保存标签页 | 点击后成功保存，过滤系统页面，去重正常，关闭原标签页，打开CoreTab页面 |
| F2 日期分组展示 | 正确按今天/昨天/更早分组，每组显示数量和时间 |
| F3 恢复单个 | 点击后在当前窗口打开新标签 |
| F4 恢复整组 | 点击后在新窗口打开该组所有标签页 |
| F5 删除单个 | 点击后从列表中移除，无残留 |
| F6 删除整组 | 通过菜单删除整组，需要确认 |

| 场景 | 处理方式 |
|------|----------|
| 无标签页可保存 | 提示"没有打开的标签页" |
| 存储空间不足 | 提示存储空间不足 |
| 网络问题(同步) | 提示网络错误，自动重试 |
